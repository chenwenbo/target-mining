## 2. 产品定位与目标

摸排小程序面向摸排人员/街道账号，用于移动端完成企业摸排任务执行、企业信息查看、走访记录提交、企业测评辅助、个人走访统计和账号管理。

核心目标：

| 目标 | 说明 |
| --- | --- |
| 任务闭环 | 支持账号登录后查看本人或管辖区域任务，按状态筛选、搜索并进入企业详情。 |
| 现场走访 | 支持三步走访表单、草稿保存、预览提交和任务状态同步。 |
| 企业测评 | 支持生成企业自主测评链接，也支持工作人员现场代填测评并同步管理端。 |
| 过程统计 | 支持按当前登录人员或账号范围展示走访 KPI、意愿分布和任务状态总览。 |

## 3. 使用角色

| 角色 | 使用范围 |
| --- | --- |
| 摸排人员 | 登录小程序，查看名下或可见任务，查看企业详情，提交走访记录，生成/代填测评，查看个人统计和个人信息。 |
| 街道账号 | 登录小程序，查看管辖区域或负责人名下任务，完成与摸排人员一致的走访、测评和统计操作。 |

## 6. 核心业务流程

### 6.1 登录与任务查看

1. 摸排人员/街道账号输入用户名和密码登录。
2. 系统校验账号是否已启用。
3. 登录成功后进入“我的任务”。
4. 系统按街道优先隔离任务，并兼容按经办人过滤。
5. 用户可查看待走访/超期统计，按状态切换或按企业名称搜索。
6. 点击任务卡片进入对应企业详情。

### 6.2 走访记录提交

1. 用户从未完成任务的企业详情页点击“开始走访”。
2. 系统进入三步走访表单：走访信息、情况核实、申报意愿。
3. 用户可保存草稿，草稿使任务生命周期进入“摸排中”。
4. 表单完成后进入提交前预览，可返回修改或确认提交。
5. 确认提交后保存走访记录，任务状态更新为“进行中”，PC 端可见记录。

### 6.3 企业测评

| 流程 | 步骤 |
| --- | --- |
| 企业自主测评 | 生成绑定当前企业和任务的专属链接；显示分享链接；支持复制链接和二维码；扫码打开公开测评页；必要时重新生成并覆盖当前待完成记录。 |
| 工作人员代填 | 从代填入口进入问卷；按研发投入、知识产权、收入结构与管理三步填写；每步题目必答；自动保存草稿；预览后确认提交；提交后展示得分、等级、维度得分并同步管理端。 |

## 8. 权限与数据规则

| 规则类型 | 规则 |
| --- | --- |
| 登录权限 | 仅已启用账号可登录；登录失败需提示错误。 |
| 任务可见范围 | 任务列表展示当前账号管辖区域或负责人名下任务；优先按街道隔离，兼容按经办人过滤。 |
| 统计范围 | 走访 KPI 按当前登录人员统计；任务总览和我的数据按当前账号任务范围统计。 |
| 任务超期 | 未完成且截止时间早于当前日期的任务定义为超期。 |
| 任务生命周期 | 保存走访草稿后，任务管理可识别为“摸排中”；提交走访记录后任务状态更新为“进行中”；已完成任务不展示开始走访按钮，展示任务已完成状态。 |
| 企业详情数据 | 企业详情需包含企业概况、走访历史、企业测评等信息；走访历史按提交时间倒序展示。 |
| 知识产权明细 | 发明专利、实用新型、外观设计、软著数量为 0 的类别不可点击；有数据时以底部弹层展示明细。 |
| 企业状态标签 | 经营异常、行政处罚、已认定高企、科技型中小企业状态根据企业风险和入库字段判断颜色。 |
| 内部备注 | 工作人员内部备注不对企业展示，仅内部可见。 |
| 测评链接 | 企业自主测评链接绑定当前企业和任务；重新生成后新链接覆盖当前待完成记录。 |
| 数据同步 | 走访记录提交后 PC 端可见；代填测评提交后同步到管理端，PC 企业详情可看到测评结果。 |

## 9. 非功能需求

| 类别 | 需求 |
| --- | --- |
| 易用性 | 任务列表需支持状态 Tab、实时搜索和关键数量展示；三步表单和三步测评均需展示进度与当前步骤。 |
| 移动端交互 | 企业联系人电话支持手机端唤起拨号；测评链接支持复制反馈和二维码开关显示。 |
| 数据一致性 | 列表、详情、预览页、提交后详情、PC 端统计/详情之间的数据需一致。 |
| 校验反馈 | 必填项、字数限制、时长范围、必答题未完成等场景需阻断下一步或提交，并给出明确提示。 |
| 可追溯性 | 走访历史需按提交时间倒序展示，记录数量和内容与提交记录一致。 |
| 待确认 | 表格未明确性能指标、兼容设备范围、安全审计、异常重试、接口超时和日志要求，建议后续补充。 |
