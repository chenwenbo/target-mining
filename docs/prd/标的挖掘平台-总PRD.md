## 1. 产品定位

标的挖掘平台面向高企培育、标的筛选、企业摸排、资质测评和平台运营场景，帮助政府侧租户从企业池中识别潜在高企标的，派发摸排任务，收集走访与测评数据，并通过统计驾驶舱、智能体问答和运营后台完成业务闭环。

平台由 5 个系统端组成：

| 系统端 | 核心用户 | 定位 |
| --- | --- | --- |
| 政府PC端 | 租户用户 | 租户侧业务主工作台，负责标的池筛选、任务派发、企业详情、摸排统计、测评管理和智能体问答。 |
| 摸排小程序 | 摸排人员/街道账号 | 一线走访执行端，负责查看任务、记录走访、生成或代填测评、查看个人统计。 |
| 企业公开测评端 | 企业联系人 | 企业通过专属链接完成高企资质测评并查看结果建议。 |
| 运营后台 | 平台运营 | 平台侧租户、账号、概览和开通管理后台。 |
| 获客落地页 | 访客 | 面向潜在客户的街道/乡镇标的分析、名单解锁和演示申请入口。 |

## 2. 产品目标

1. 形成从潜在客户获客、租户开通、标的筛选、任务派发、实地摸排、企业测评到统计分析的业务闭环。
2. 提升政府侧租户对潜在高企标的的发现、跟进和转化效率。
3. 降低一线摸排人员记录走访和回传数据的操作成本。
4. 将企业自主测评与工作人员代填测评结果同步回管理端，为培育建议和后续行动提供依据。
5. 让平台运营能够集中管理租户、账号、开通状态和平台运行概览。

成功指标在源表中未定义，建议后续补充：标的派发完成率、走访完成率、测评完成率、有申报意愿企业转化率、租户活跃率、线索提交转化率。

## 5. 核心业务流程

### 5.1 获客到租户开通

1. 访客在获客落地页选择街道/乡镇并发起潜在高企分析。
2. 页面展示企业总数、泛科技企业、高潜力企业和前三条预览企业；未解锁名单展示模糊行。
3. 访客通过“解锁完整名单”或“申请免费演示”提交姓名、单位、手机号。
4. 平台运营在运营后台创建租户、设置管理员账号、维护租户到期日与状态。
5. 租户管理员获得账号后登录政府PC端开展标的筛选与摸排业务。

### 5.2 标的筛选与任务派发

1. 租户用户进入政府PC端，查看驾驶舱、企业漏斗、年度目标和标的池。
2. 在标的池按企业名称、信用代码、技术领域、街道/乡镇、申报类型、申报意愿等筛选企业。
3. 用户可对当前筛选或已选企业导出 CSV。
4. 用户对单个企业或批量企业进行派发，选择经办人并填写备注。
5. 系统在任务管理生成待走访任务，小程序端按街道或经办人范围展示任务。

### 5.3 走访执行与数据回传

1. 摸排人员登录小程序后进入“我的任务”，查看待走访、超期、进行中和已完成任务。
2. 摸排人员进入企业详情，查看企业概况、知识产权、合规状态和走访历史。
3. 摸排人员启动三步走访表单，录入走访信息、情况核实和申报意愿。
4. 保存草稿时任务生命周期进入“摸排中”；提交后走访记录同步到政府PC端企业详情和摸排统计。
5. 政府PC端按租户范围聚合走访 KPI、意愿分布、街道对比、障碍关键词和企业需求。

### 5.4 企业测评闭环

1. 政府PC端或摸排小程序可为企业生成自主测评专属链接和二维码。
2. 企业联系人通过公开测评端访问问卷，按研发投入、知识产权、收入结构与管理三步完成单选题。
3. 企业提交后测评记录状态变为已完成，填写方式为企业自填，并跳转结果页。
4. 摸排人员也可在小程序端代填测评，提交后同步到管理端。
5. 政府PC端企业详情展示最新完成测评记录、综合得分、等级、维度得分、答题详情、AI 分析和可打印报告。

### 5.5 平台运营与账号管理

1. 平台运营登录运营后台查看租户、企业、任务等平台级 KPI。
2. 运营人员维护租户基础信息、管理员账号、到期日、状态和备注。
3. 运营人员可在账号开通模块查看、复制、批量复制或导出租户管理员凭证。
4. 租户用户在政府PC端维护摸排小程序账号，进行新建、启停、重置密码、复制凭证和导出。
5. 禁用的租户账号或摸排账号不得继续登录对应系统。

## 7. 权限与访问控制

| 角色 | 可访问端 | 权限范围 |
| --- | --- | --- |
| 平台运营 | 运营后台 | 查看平台概览，管理租户、租户详情、租户账号开通和凭证。 |
| 租户用户 | 政府PC端 | 查看租户范围内标的、任务、企业详情、摸排统计、测评记录和摸排小程序账号。 |
| 摸排人员/街道账号 | 摸排小程序 | 查看当前账号管辖区域或负责人名下任务，提交走访和测评。 |
| 企业联系人 | 企业公开测评端 | 通过专属链接访问并提交对应企业测评，查看测评结果与建议。 |
| 访客 | 获客落地页 | 查看街道/乡镇潜在高企分析预览，提交名单解锁或演示申请。 |

全局规则：

1. 禁用账号不得登录对应系统。
2. 退出登录后访问受保护页面必须重新进入登录校验。
3. 小程序任务范围优先按街道隔离，并兼容按经办人过滤。
4. 政府PC端租户用户重置筛选后仍需保留本街道锁定规则。
5. 测评链接通过 token 匹配待完成测评记录，无有效测评结果时不允许查看结果页。

## 8. 优先级与发布建议

### 8.1 P1 范围

P1 共 106 个功能点，应作为首版验收主范围，覆盖登录、标的池、派发、走访、测评、统计、租户管理和获客提交等核心闭环。

首版建议包含：

1. 运营后台租户创建、账号开通和启停。
2. 政府PC端登录、驾驶舱、标的池、企业详情、任务派发、摸排统计和小程序账号管理。
3. 摸排小程序登录、任务查看、企业详情、走访提交、代填测评和个人统计。
4. 企业公开测评端链接访问、问卷填写、提交和结果查看。
5. 获客落地页街道分析、名单解锁和演示申请。

### 8.2 P2 范围

P2 共 34 个功能点，建议纳入增强版本或首版补充范围，包括快捷指令、历史对话、草稿、导出/复制增强、测评报告、培育建议和部分图表能力。

### 8.3 P3 范围

P3 共 1 个功能点，为政府PC端高企测评管理的测评等级/得分分布。该功能依赖足够的已完成测评记录，建议在测评数据积累后排期。

## 9. 全局非功能需求

| 类别 | 需求 |
| --- | --- |
| 可用性 | PC 端列表、看板、详情、图表和小程序表单需保证字段完整、状态清晰、错误反馈明确。 |
| 数据一致性 | 任务、走访记录、测评记录、账号状态在各端展示需保持一致。 |
| 可追溯性 | 走访记录、测评记录、账号重置、租户状态变更建议保留操作时间和操作者，源表未定义具体审计要求，需待确认。 |
| 安全性 | 密码默认隐藏；复制、重置、删除等敏感操作需明确反馈，删除和重置使用二次确认。 |
| 导出能力 | 标的池、摸排账号、租户账号等 CSV 导出需保证中文正常、字段完整、范围符合当前筛选或选择。 |
| 兼容性 | 小程序端需适配移动端操作；公开测评链接和二维码需在移动端可访问。 |
| 异常处理 | 登录失败、无数据、无测评结果、无专利记录、不存在租户等场景需有明确空态或错误提示。 |

## 10. 全局验收标准

1. 141 个源表功能点均在对应端级 PRD 中有明确需求描述、规则或验收口径。
2. P1 功能均可完成主流程联调：租户开通、政府PC登录、标的筛选派发、小程序走访、测评提交、PC 端结果查看。
3. 禁用账号、错误登录、退出后访问保护页、无数据、无结果等异常场景均符合源表规则。
4. 跨端同步正确：派发任务可在小程序看到；提交走访后 PC 可查看记录和统计；提交测评后 PC 可查看结果。
5. 导出、复制、二维码、拨号、tel/mailto 等交互按源表验收点完成。
6. 运营后台租户状态、管理员账号状态与政府PC端登录能力一致。
7. 未挂菜单功能不进入正式菜单，除非产品确认正式入口。
