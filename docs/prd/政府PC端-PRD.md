## 2. 产品定位与目标

政府PC端面向租户用户，支撑政府侧围绕高企申报和标的挖掘进行企业筛选、派发摸排、企业画像查看、专业测评、任务跟踪、统计分析及摸排小程序账号管理。

核心目标：

| 目标 | 说明 |
| --- | --- |
| 标的发现与管理 | 通过泛科技企业、潜在标的企业、有意愿企业等层级管理企业池，支持搜索、筛选、排序、导出和派发。 |
| 摸排任务闭环 | 从标的池或任务看板派发企业，形成待走访任务，并通过任务看板和摸排统计跟踪执行结果。 |
| 企业信息支撑 | 在企业详情中集中展示工商信息、知识产权、专利、著作权、商标、备案、测评和摸排记录。 |
| 决策统计分析 | 通过驾驶舱和摸排统计查看年度目标、申报进度、企业漏斗、街道/乡镇、技术领域、意愿分布、障碍和需求。 |
| 智能辅助 | 提供智能体问答、快捷指令、历史对话和 AI 分析能力，辅助用户分析标的、任务和统计数据。 |

## 6. 核心业务流程

### 6.1 登录与访问

1. 租户用户输入账号密码。
2. 系统先识别租户用户，再识别摸排账号。
3. 禁用账号不得登录。
4. 租户用户登录成功后进入标的池视图。
5. 用户可在个人中心修改密码，或退出登录并返回登录页。

### 6.2 标的筛选与任务派发

1. 用户进入标的池，在泛科技企业、潜在标的企业、有意愿企业之间切换。
2. 用户通过企业名称/统一社会信用代码搜索，并按技术领域、街道/乡镇、申报类型、申报意愿等条件筛选。
3. 用户可排序、勾选单个企业、全选当前筛选结果，并导出 CSV。
4. 用户对单个或批量企业发起派发，选择经办人并填写备注。
5. 确认后系统在任务管理中生成待走访任务。
6. 有意愿企业池中派发按钮禁用，不允许重复派发。

### 6.3 任务看板与摸排统计

1. 用户在任务管理查看标的池、已派发待摸排、已完成三列看板。
2. 用户可从标的池拖拽企业到已派发列，确认经办人后生成待走访任务。
3. 用户按企业名称、经办单位、经办人、申报意愿筛选任务，或清除筛选恢复默认租户范围。
4. 摸排数据形成统计概览、意愿分布、街道对比、高频障碍和企业需求统计。
5. 点击数据刷新后重新读取最新摸排数据并重新聚合。

### 6.4 企业详情与测评

1. 用户从标的池进入企业详情页，可在企业画像、专业测评、企业摸排 Tab 间切换。
2. 企业画像展示工商信息、知识产权、专利、著作权、商标、备案等信息。
3. 专业测评展示最新测评状态；无记录时提示发起测评。
4. 用户可生成/重新生成测评链接，复制链接并展示二维码。
5. 测评完成后展示结果总览、答题详情、AI 智能分析，并支持导出/打印测评报告。
6. 企业摸排 Tab 展示该企业历次走访摸排记录。

### 6.5 智能体辅助

1. 用户在智能体首页查看能力说明、推荐指令和提问输入入口。
2. 输入问题后进入对话页，展示用户消息、思考状态和助手回复。
3. 智能体基于标的、任务、统计数据返回回答。
4. 用户可点击快捷指令填充输入框，也可恢复历史对话或新建对话返回首页。

## 8. 权限与数据规则

| 类型 | 规则 |
| --- | --- |
| 登录识别 | 系统先识别租户用户，再识别摸排账号；禁用账号不允许登录。 |
| 受保护页面 | 退出后受保护页面不可继续访问，访问业务页面需重新登录校验。 |
| 租户范围 | 摸排统计按当前租户范围统计；任务清除筛选后恢复默认租户范围。 |
| 街道/乡镇 | 标的池支持街道/乡镇多选筛选；筛选重置后租户用户仍保留本街道锁定。 |
| 标的层级 | 泛科技企业、潜在标的企业、有意愿企业使用不同过滤规则。 |
| 派发限制 | 有意愿企业不允许重复派发；有意愿企业池中派发按钮禁用。 |
| 数据聚合 | 驾驶舱图表、摸排统计图表均基于当前标的、任务、走访/测评数据聚合。 |
| 测评记录 | 企业详情专业测评优先展示最新完成记录；结果、答题详情、AI 分析按最新完成测评记录展示。 |
| 账号状态 | 禁用摸排账号不能登录小程序或参与账号切换。 |
| 本地保存 | 年度目标编辑值保存到本地；智能体历史对话本地保存标题、创建时间、更新时间和消息列表。具体持久化方案待确认。 |

## 9. 非功能需求

| 类别 | 需求 |
| --- | --- |
| 易用性 | 搜索、筛选、排序、批量选择、导出、派发、清除筛选等操作应提供明确状态反馈。 |
| 可读性 | 驾驶舱、统计图表、漏斗、柱状图、饼图需数值清晰可读；详情页表格字段完整、排版清晰。 |
| 数据一致性 | KPI、图表、列表、详情、任务和统计数据需与对应数据源保持一致。 |
| 安全性 | 密码修改需校验当前密码、新旧密码、两次确认；退出后业务页面需重新校验；账号删除、重置密码需二次确认。 |
| 导出能力 | 标的池导出 CSV、摸排账号导出 CSV、测评报告导出/打印需保证字段完整、中文正常、内容正确。 |
| 兼容性 | 电话和邮箱链接分别使用 `tel:` / `mailto:`；二维码需可扫码打开。 |
| 待确认/建议 | 本地存储数据、权限粒度、图表刷新频率、CSV 字段顺序、测评报告模板、AI 分析生成策略、异常/空状态标准需进一步确认。 |
