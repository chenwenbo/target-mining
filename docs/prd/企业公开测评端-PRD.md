## 2. 产品定位与目标

企业公开测评端用于企业联系人通过专属链接完成高企资质测评问卷，并查看测评结果与高企培育建议。

### 2.1 产品目标

| 目标 | 说明 |
| --- | --- |
| 支持企业自助进入测评 | 企业联系人通过专属链接进入对应企业的高企资质测评问卷。 |
| 支持完整问卷作答与提交 | 展示企业名称、测评分组、题目选项，支持单选答题、步骤切换和提交。 |
| 支持结果与建议展示 | 完成测评后展示测评得分、等级、维度得分及培育建议。 |

### 2.2 边界说明

| 类型 | 范围 |
| --- | --- |
| 本期明确范围 | 测评链接访问、问卷展示、单选答题、步骤切换、测评提交、结果展示、培育建议展示。 |
| 待确认/建议 | 链接失效处理、未完成测评继续作答、题目配置来源、分数和等级算法、建议生成规则、移动端适配标准、埋点与通知机制。 |

## 6. 核心业务流程

### 6.1 测评作答流程

1. 企业联系人打开专属测评链接。
2. 系统根据 token 匹配待完成测评记录。
3. 链接有效时，系统打开对应企业问卷。
4. 系统展示企业名称、测评分组和题目选项。
5. 企业联系人按步骤完成单选题作答。
6. 企业联系人可在步骤间前进或返回，返回时已选答案不丢失。
7. 企业联系人提交问卷。
8. 系统将记录状态改为已完成，填写方式标记为企业自填。
9. 系统跳转结果页，并同步 PC 端测评结果。

### 6.2 结果查看流程

1. 企业联系人完成测评后进入结果页。
2. 系统展示企业测评得分、等级和维度得分。
3. 系统展示基于测评结果生成的高企培育建议。
4. 重要建议与普通建议需做视觉区分。
5. 无测评结果时不允许查看结果页。

## 9. 非功能需求

| 类型 | 需求 |
| --- | --- |
| 可用性 | 问卷步骤切换后已选答案需保留；选中项需高亮，便于企业联系人确认答题状态。 |
| 一致性 | 企业名称、题目、答案状态、测评结果需与对应企业和测评记录一致。 |
| 数据同步 | 提交后企业公开测评端与 PC 端测评结果需保持同步。 |
| 异常处理 | token 无效、无测评结果、提交失败、网络异常等提示文案与恢复方式待确认。 |
| 兼容性 | 公开测评端的设备与浏览器适配范围待确认/建议明确。 |
| 安全性 | 专属链接 token 的有效期、加密方式、访问次数限制待确认/建议明确。 |
